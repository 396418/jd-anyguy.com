Was ist ein Dateisystem?

Ein Dateisystem organisiert Daten auf einem Datenträger.

  • Speicherung von Dateien
  • Verwaltung von Verzeichnissen
  • Zuordnung von Speicherblöcken
  • Verwaltung von Zugriffsrechten

Linux-Verzeichnisstruktur

Verzeichnis Bedeutung
/ Root-Verzeichnis
/home Benutzerdateien
/etc Konfigurationsdateien
/var Logs und variable Daten
/mnt Temporäre Mountpunkte

Filesystem Hierarchy Standard

Der FHS legt fest, wie Linux-Verzeichnisse strukturiert sein sollen.

  • einheitlicher Aufbau
  • leichtere Administration
  • Programme finden Dateien an erwarteten Orten

Wichtige Dateisystemtypen

ext4

  • weit verbreitet
  • stabil
  • Standard bei vielen Linux-Systemen

XFS

  • gute Performance
  • oft im Serverbereich
  • gut für große Datenmengen

Btrfs

  • modernes Dateisystem
  • Snapshots möglich
  • mehr Funktionen, aber komplexer

Das Mounting-Konzept

Linux verwendet keine Laufwerksbuchstaben wie Windows.

Datenträger werden in den Verzeichnisbaum eingebunden.

Das nennt man Mounten.

Das Mounting-Konzept

Beispiel für einen Mountpunkt:

/mnt/daten

Danach ist das Dateisystem über diesen Pfad erreichbar.

Wichtige Linux-Befehle

lsblk
df -h
mount
umount
blkid
  • lsblk → Geräte und Partitionen anzeigen
  • df -h → Speicherplatz anzeigen
  • mount → Dateisystem einhängen
  • umount → Dateisystem aushängen
  • blkid → UUID und Dateisystemtyp anzeigen

Beispiel: manuelles Mounten

sudo mkdir /mnt/daten
sudo mount /dev/sdb1 /mnt/daten
ls /mnt/daten
sudo umount /mnt/daten

Zuerst wird ein Mountpunkt erstellt.

Danach wird die Partition dort eingebunden.

/etc/fstab

Die Datei /etc/fstab legt fest, welche Dateisysteme beim Start automatisch gemountet werden.

/etc/fstab

UUID=1234-5678  /mnt/daten  ext4  defaults  0  2

UUID = eindeutige Gerätekennung

/mnt/daten = Mountpunkt

ext4 = Dateisystemtyp

Warum UUID statt /dev/sdb1?

  • Gerätenamen können sich ändern
  • UUID ist eindeutig und stabil
  • dadurch weniger Fehler beim Booten

Praxisdemo auf dem Raspberry Pi

lsblk
lsblk -f
df -h
mount
cat /etc/fstab

Damit zeige ich die Geräte, Dateisysteme und Mountpunkte auf einem echten Linux-System.

Typische Fehler

  • falscher Mountpunkt
  • Tippfehler in /etc/fstab
  • falscher Dateisystemtyp
  • Partition ist noch in Benutzung

Fazit

  • Linux nutzt einen einheitlichen Verzeichnisbaum
  • Dateisysteme organisieren Daten auf Datenträgern
  • Datenträger werden gemountet
  • /etc/fstab automatisiert das Mounten
  • Das Thema ist zentral für Linux-Administration
