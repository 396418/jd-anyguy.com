IT & Netzwerk Glossar

A

APT – Advanced Packaging Tool

Linux Paketverwaltung

Erklärung: APT ist ein Tool unter Linux (Debian/Ubuntu), das Software installiert, aktualisiert und verwaltet.

Beispiel: sudo apt install firefox installiert den Firefox Browser.

ASCII – American Standard Code for Information Interchange

Zeichenkodierung

Erklärung: ASCII ist ein Standard, der Zeichen wie Buchstaben und Zahlen in Zahlenwerte umwandelt.

Beispiel: Der Buchstabe „A“ hat den Wert 65.

Attribut

Datenbank

Erklärung: Ein Attribut beschreibt eine Eigenschaft einer Entität in einer Datenbank.

Beispiel: Name oder Geburtsdatum einer Person.

B

BCC – Blind Carbon Copy

E-Mail

Erklärung: BCC ist eine versteckte E-Mail-Kopie, die andere Empfänger nicht sehen können.

Beispiel: Newsletter an viele Empfänger ohne Sichtbarkeit der Adressen.

BIND – Berkeley Internet Name Domain

DNS Server

Erklärung: BIND ist eine Software zur Verwaltung von DNS-Servern.

Beispiel: Auflösung von www.google.de in eine IP-Adresse.

BSI – Bundesamt für Sicherheit in der Informationstechnik

IT Sicherheit

Erklärung: Das BSI ist eine deutsche Behörde für Cyber-Sicherheit.

Beispiel: Warnungen vor Sicherheitslücken.

C

CC – Carbon Copy

E-Mail

Erklärung: CC ist eine sichtbare Kopie einer E-Mail an weitere Empfänger.

Beispiel: Alle Teammitglieder sehen die Mail.

CMS – Content Management System

Webseitenverwaltung

Erklärung: CMS ermöglicht das Erstellen und Verwalten von Webseiten ohne Programmierung.

Beispiel: WordPress.

Constraint

Datenbankregel

Erklärung: Ein Constraint ist eine Regel, die festlegt, welche Daten erlaubt sind.

Beispiel: Ein Primärschlüssel muss eindeutig sein.

CUPS – Common UNIX Printing System

Drucksystem

Erklärung: CUPS verwaltet Druckaufträge unter Linux/Unix-Systemen.

Beispiel: Netzwerkdrucker im Büro.

D

Daemon

Hintergrundprozess

Erklärung: Ein Daemon ist ein Programm, das im Hintergrund automatisch läuft.

Beispiel: sshd für SSH-Verbindungen.

DDoS – Distributed Denial of Service

Angriff

Erklärung: Viele Computer überlasten gemeinsam einen Server.

Beispiel: Website ist nicht erreichbar.

DNS – Domain Name System

Namensauflösung

Erklärung: DNS übersetzt Domains in IP-Adressen.

Beispiel: google.de → IP-Adresse.

DoS – Denial of Service

Angriff

Erklärung: Ein Server wird durch viele Anfragen überlastet.

Beispiel: Webseite reagiert nicht mehr.

H

HTTP – Hypertext Transfer Protocol

Webprotokoll

Erklärung: HTTP überträgt Webseiten im Internet.

Beispiel: http://example.com

I

IDE – Integrated Development Environment

Entwicklungsumgebung

Erklärung: Software zum Programmieren und Testen von Code.

Beispiel: Visual Studio Code.

IP – Internet Protocol

Netzwerkadresse

Erklärung: IP identifiziert Geräte im Netzwerk.

Beispiel: 192.168.0.1

IPP – Internet Printing Protocol

Drucken

Erklärung: Protokoll zum Drucken über Netzwerke.

Beispiel: WLAN Drucker.

L

LAMP

Webserver Stack

Erklärung: Kombination aus Linux, Apache, MySQL und PHP.

Beispiel: Webseiten Hosting.

LPI – Linux Professional Institute

Zertifizierung

Erklärung: Organisation für Linux-Zertifikate.

Beispiel: LPIC-1 Prüfung.

LWL – Lichtwellenleiter

Glasfaser

Erklärung: Datenübertragung mit Licht in Glasfaserkabeln.

Beispiel: Highspeed Internet.

M

MAC – Media Access Control

Hardwareadresse

Erklärung: Eindeutige Adresse einer Netzwerkkarte.

Beispiel: 00:1A:2B:3C:4D:5E

N

NAT – Network Address Translation

Netzwerk

Erklärung: Mehrere Geräte teilen eine öffentliche IP.

Beispiel: Heimrouter.

NIC – Network Interface Card

Netzwerkkarte

Erklärung: Hardware für Netzwerkverbindung.

Beispiel: WLAN Adapter.

NTP – Network Time Protocol

Zeit

Erklärung: Synchronisiert Computerzeit.

Beispiel: automatische Uhrzeit.

P

Port

Netzwerk

Erklärung: Identifiziert Dienste auf einem Gerät.

Beispiel: Port 80 für HTTP.

proprietär

Software

Erklärung: Quellcode ist nicht öffentlich.

Beispiel: Microsoft Windows.

pwd

Terminal

Erklärung: Zeigt aktuelles Verzeichnis an.

Beispiel: /home/user

R

RFC – Request for Comments

Internetstandard

Erklärung: Dokumente für Internetprotokolle.

Beispiel: HTTP Standard.

S

scp – Secure Copy

Dateiübertragung

Erklärung: Sicheres Kopieren über SSH.

Beispiel: scp file server:/

sed

Textverarbeitung

Erklärung: Automatische Bearbeitung von Text.

Beispiel: Wörter ersetzen.

SFTP

Dateitransfer

Erklärung: Sicherer Dateiübertragungsdienst.

Beispiel: Upload auf Server.

SQL – Structured Query Language

Datenbank

Erklärung: Sprache für Datenbankabfragen.

Beispiel: SELECT * FROM users;

Source

Quelle

Erklärung: Ursprung von Daten oder Code.

Beispiel: Quellcode

T

Target

Ziel

Erklärung: Ziel einer Operation.

Beispiel: Zielordner

TCP – Transmission Control Protocol

Netzwerk

Erklärung: Zuverlässige Datenübertragung.

Beispiel: Webseiten.

Tupel

Datenbank

Erklärung: Eine Zeile in einer Tabelle.

Beispiel: Datensatz eines Nutzers.

U

Ubiquität

Verfügbarkeit

Erklärung: Von überall zugänglich.

Beispiel: Cloud Dienste.

UDP – User Datagram Protocol

Netzwerk

Erklärung: Schnelle, aber unzuverlässige Übertragung.

Beispiel: Online Gaming.

UML

Modellierung

Erklärung: Darstellung von Systemen.

Beispiel: Klassendiagramm.

URL – Uniform Resource Locator

Webadresse

Erklärung: Adresse im Internet.

Beispiel: https://google.com

UTC

Weltzeit

Erklärung: Standardisierte Weltzeit.

Beispiel: UTC+1 Deutschland

V

Verbose

Ausgabe

Erklärung: Detaillierte Ausgabe im Terminal.

Beispiel: ping -v

VCS – Version Control System

Versionskontrolle

Erklärung: Verwaltung von Code-Versionen.

Beispiel: Git

VLAN

Netzwerk

Erklärung: Virtuelle Netzwerke.

Beispiel: getrennte Abteilungen.

VPN – Virtual Private Network

Sicherheit

Erklärung: Verschlüsselte Verbindung über Internet.

Beispiel: Firmenzugang.

W

WLL – Wireless Local Loop

Funkinternet

Erklärung: Drahtloser Internetzugang.

Beispiel: Internet über Funk.
